Highlighted options

core options

Najczęściej używane opcje konfiguracji flow. Zaznaczone tak, żeby od razu było jasne co kontroluje przebieg runtime.

strictSchema type: boolean | default: true

Wymusza walidację wszystkich kroków względem schema przed uruchomieniem flow.

previewParity type: "strict" | "relaxed"

Definiuje próg zgodności między preview a wykonaniem w terminalu.

runtimeTarget type: "bash" | "zsh" | "pwsh"

Wybór środowiska shell, w którym flow renderuje output końcowy.

trace type: "off" | "errors" | "verbose"

Poziom diagnostyki i logowania decyzji warunków podczas wykonania.

Flow runtime API

reference

Minimalny zestaw endpointów i payloadów do uruchamiania flow, synchronizacji preview i odbioru wyników.

Method Endpoint Description Auth
POST /v1/flows/validate Waliduje YAML i zwraca listę błędów z pozycją w schema. token
POST /v1/flows/preview Renderuje snapshot TUI dla aktualnego stanu danych wejściowych. token
POST /v1/flows/run Uruchamia flow runtime i streamuje zdarzenia kroków. token
GET /v1/runs/:id/state Zwraca status i aktualny krok w wykonywanym runie. token
GET /v1/runs/:id/output Pobiera finalny payload output po zakończeniu flow. token
Example / YAML flow definition init-project.yaml 
flow: init-project
version: 1

steps:
  - id: project_name
    type: input
    label: Project Name
    target: project.name

  - id: framework
    type: select
    label: Select Framework
    when: project_name != ''
    options: [react, vue, svelte]

output:
  target: config.framework
  onSuccess: print('Ready to scaffold')

# preview parity contract: strict
Example / TypeScript client call runtime 
const run = await steply.flows.run({
  file: "init-project.yaml",
  mode: "strict",
  shell: "bash",
  context: {
    actor: "cli",
    workspace: "/repo/project"
  }
})

if (run.previewParity < 0.98) {
  throw new Error("Preview mismatch")
}

Variables and context map

important

Każda zmienna ma przewidywalny lifecycle: input, transform, runtime output. Poniżej najważniejsze sloty danych.

$input.*

Wartości bezpośrednio z widgetów (np. $input.project_name), dostępne od razu po zatwierdzeniu kroku.

$context.*

Metadane środowiska uruchomienia: shell, workspace, actor, run id, timestamp.

$runtime.*

Dane obliczane podczas wykonania: status tasków, trace, warningi i referencje do output map.

$output.*

Finalny payload przekazywany dalej do pipeline, CI lub lokalnego skryptu bootstrapującego.

Variable Type Available Example
$input.project_name string after step 1 "my-app"
$input.framework string after select step "react"
$context.shell enum always "bash"
$runtime.current_step number during run 2
$output.config.framework string on success "react"

State and error model

runtime states

Steply rozdziela problemy walidacji schema od błędów runtime, żeby diagnoza była szybka i jednoznaczna.

READY - flow validated and waiting for input
BLOCKED - condition when not satisfied
PARTIAL - optional task failed, run continues
FAILED - runtime task error, run aborted
Error payload shape json 
{
  "code": "SCHEMA_CONDITION_INVALID",
  "message": "Unknown variable in 'when' expression",
  "step": "framework",
  "line": 28,
  "hint": "Use $input.project_name"
}
