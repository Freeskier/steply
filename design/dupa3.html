Why Steply

Not another prompt wrapper.
A real terminal product engine.

01
architecture

One engine, multiple entry points

Write a YAML flow, compose a draft flow from shell, or preview the same UI in the browser. The rendering model stays aligned.

02
runtime

Terminal UX with structure

Steply is not just a prompt picker. It gives you steps, validation, task hooks, embedded widgets, state, and a real interaction model.

03
preview

Browser tooling without a fake preview

The web preview is not a hand-made approximation. It is powered by the same Rust/WASM rendering path that shapes the terminal runtime.

Three ways to use it

Same engine.
Different entry point.

flow.yaml 
steps:
  - id: user
    title: Basic user data
    widgets:
      - type: text_input
        id: name
        label: Name
      - type: text_input
        id: email
        label: Email
  - id: project
    title: Project setup
    skip_if: "env.SKIP_PROJECT == '1'"
    widgets:
      - type: file_browser
        id: project_dir
        label: Project directory
      - type: select
        id: framework
        label: Framework
        options: [React, Vue, Svelte, Solid]

How a flow comes together

From config to terminal.
Four stages.

01

Define

Start from YAML or compose a draft flow from shell commands. Both paths produce the same config model.

flow.yaml 
steps:
  - id: user
    title: Setup
    widgets:
      - type: text_input
        id: name
02

Preview

Render the exact interaction model in the browser and iterate before shipping. What you see is what the terminal runs.

browser preview 
┌─ Setup ──────────────────┐
│  Name ▮                  │
│                          │
│  [→ Continue]            │
└──────────────────────────┘
03

Run

Launch the flow in the terminal with real focus management, validation, state, and task hooks.

terminal 
$ steply run flow.yaml

  Setup
  ─────
  Name: Jordan Lee
  Email: jordan@ex.com

  ✓ Completed
04

Scale

Generate docs, expose examples, and move toward a visual creator without rewriting the product model.
