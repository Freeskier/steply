Built for predictable terminal UX.

Produkt nie zgaduje. Authoring, warunki i render opierają się na jawnych schema oraz spójnym modelu runtime, dzięki czemu flow zachowuje się identycznie w creatorze i finalnym shellu.

Schema-aware authoring

Walidacja struktur YAML i konfiguracji widgetów na etapie projektowania, bez ręcznego debugowania po uruchomieniu.

Composable runtime API

Flow możesz składać z mniejszych bloków i mapować ich wynik bez utraty kontroli nad typami i kolejnością kroków.

Preview parity

To co widzisz w web preview, to to samo co renderuje się w terminalu podczas wykonania komendy w runtime.

Workflow in four steps.

Od szablonu do uruchomienia: krótka ścieżka, z minimalną liczbą miejsc gdzie konfiguracja może się rozjechać.

01 / template

Start from template

Wybierz bazowy flow i wygeneruj szkic YAML z poprawną strukturą wejść i outputów.

02 / edit

Edit widgets

Konfiguruj pola, etykiety i mapowania targetów, jednocześnie widząc status walidacji.

03 / logic

Conditions and tasks

Dodawaj reguły `when`, sterowanie kolejnością kroków i zadania wykonywane po akceptacji flow.

04 / run

Preview and run

Sprawdź interakcję w terminal preview, a następnie uruchom identyczny runtime przez CLI.
