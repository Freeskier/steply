MULTIPLE AUTHORING MODES

Write flows your way.

Visual editor, declarative YAML, bash-style scripts, or programmatic Rust API.
Same runtime, different entry points.

◧

Visual Creator

Drag widgets, configure properties, preview live. The creator validates schema as you build—no surprises at runtime.

steply create
≡

Declarative YAML

Schema-aware configuration with steps, widgets, conditions, tasks, and subscriptions. Versioned, composable, and human-readable.

steps: - id: setup widgets: - type: select label: "Framework"
▶

Bash-Style Scripts

Imperative scripting interface like gum or glow. Quick wizards without YAML overhead.

ENV=$(steply select "dev" "prod") steply confirm "Deploy to $ENV?" steply run-tasks deploy
{}

Rust API

Programmatic flow definition for complex logic, dynamic step generation, or embedding in Rust apps.

use steply::prelude::*; Flow::new() .step(Step::new("setup") .widget(Select::new("env")))
WORKFLOW

From template to running terminal.

1

Design Your Flow

Start with a template or blank flow. Add steps, configure widgets, set conditions. Use the visual creator for drag-and-drop, or write YAML directly.

flow.yaml
steps: - id: project_init title: "Initialize Project" widgets: - type: text_input label: "Project Name" target: config.name
2

Preview Live

Hit preview and see exactly how your flow renders in terminal. Pixel-perfect fidelity between design time and runtime.

$ steply preview flow.yaml
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ ▸ Initialize Project ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ Project Name: my-app█
3

Run in Terminal

Ship the YAML. The runtime takes over—same result, every time. Users navigate steps, fill widgets, trigger tasks. All from terminal.

$ steply run flow.yaml
✓ Project Name: my-app ✓ Framework: React ✓ Features: TypeScript, ESLint → Running setup tasks... [████████░░] 80%

CORE FEATURES

Schema-aware. Composable. Fast.

≡

Schema Validation

Every widget backed by typed schema. Runtime validates configuration as you build—no guessing, no runtime surprises.

⋈

Conditional Logic

Gate steps and widgets with expressions. Build dynamic flows that adapt to user input and runtime context.

⚙

Tasks & Subscriptions

Execute shell commands, parse JSON, trigger on input changes. Wire up runtime tasks with debounce and timeout control.

◉

Preview Parity

What you preview is what renders in terminal. Pixel-perfect fidelity between design time and runtime.

▦

Composable Steps

Steps are modular blocks. Chain them, gate them with conditions, inject dynamic data—runtime handles the flow graph.

→

Target Mapping

Map widget values to config paths. Build up state as users progress. Access previous values via refs and expressions.
